# Спецификация UI/UX: Фактическое расписание (`/admin/schedule/entries`)

Административная страница для просмотра и точечной правки фактических записей расписания.

Связанная модель Prisma:
- **`ScheduleEntry`**: конкретное занятие или режимное событие в определенный день и время.

Страница работает с фактическим расписанием. Она не редактирует `WeeklyScheduleTemplate`.

---

## 1. Роль страницы

Страница решает две задачи:
- показать администратору фактическое расписание в понятном разрезе;
- дать возможность точечно исправить конкретную запись `ScheduleEntry`.

Просмотр и редактирование относятся к одной странице, но могут реализовываться отдельными задачами.

---

## 2. Главный принцип отображения

Первая версия не показывает всю школу одной недельной сеткой.

Причина: при большом количестве классов, преподавателей и кабинетов общая недельная сетка становится нечитаемой.

Вместо этого администратор выбирает один разрез:
- `Группа`
- `Преподаватель`
- `Кабинет`

После выбора разреза администратор выбирает конкретный объект:
- конкретную группу;
- конкретного преподавателя;
- конкретный кабинет.

Основная сетка показывает расписание только выбранного объекта.

---

## 3. Layout

### Header
- `H1`: `Фактическое расписание`
- подзаголовок:
  - `Просматривайте расписание по группе, преподавателю или кабинету и исправляйте отдельные записи.`

### Панель управления
- переключатель периода:
  - `День`
  - `Неделя`
- выбор недели или даты;
- переключатель разреза:
  - `Группы`
  - `Преподаватели`
  - `Кабинеты`
- combobox выбранного объекта;
- поиск по объектам внутри выбранного разреза.

### Основная область
- read-only сетка расписания выбранного объекта;
- используется общий компонент отображения расписания, если он подходит для сценария;
- карточки отображают данные, нужные администратору:
  - предмет;
  - группа;
  - преподаватель;
  - кабинет;
  - время.

---

## 4. Просмотр

### Режим `Группы`

Администратор выбирает группу и видит расписание этой группы.

Используется для вопросов:
- что сегодня у группы;
- где находится группа в конкретный момент;
- какие преподаватели и кабинеты задействованы.

### Режим `Преподаватели`

Администратор выбирает преподавателя и видит его расписание.

Используется для вопросов:
- когда преподаватель занят;
- какие группы он ведет;
- где проходит занятие.

### Режим `Кабинеты`

Администратор выбирает кабинет и видит его расписание.

Используется для вопросов:
- когда кабинет занят;
- кто и что в нем проводит;
- есть ли свободные окна.

---

## 5. Редактирование записи

Редактирование запускается из карточки `ScheduleEntry`.

Паттерн:
- клик по карточке открывает `Sheet` с деталями записи;
- в `Sheet` редактируются поля конкретной записи;
- после сохранения сетка обновляется.

Редактирование не должно менять недельный шаблон.

Минимальные действия:
- изменить время;
- изменить кабинет;
- изменить преподавателя;
- удалить или отменить конкретную запись.

Сервер должен проверять:
- конфликт занятости преподавателя;
- конфликт занятости кабинета;
- конфликт расписания группы;
- соответствие преподавателя предмету, если меняется преподаватель.

---

## 6. Empty states

### Не выбран объект
- `Выберите группу, преподавателя или кабинет`
- `После выбора здесь появится расписание.`

### Нет записей в периоде
- `В выбранном периоде нет занятий`
- `Проверьте другой период или примените недельный шаблон к фактическому расписанию.`

### Ошибка загрузки
- `Не удалось загрузить расписание`
- `Повторите попытку или измените параметры просмотра.`

---

## 7. Что не входит в первую версию

- общая недельная сетка всей школы;
- массовое редактирование;
- экран замен преподавателей;
- экспорт и печать;
- редактирование `WeeklyScheduleTemplate`.
