# Спецификация UI/UX: Доступность преподавателей (Availability & Schedule)

**Связанные модели Prisma:** `Teacher`, `TeacherAvailability`, `TeacherAvailabilityOverride`, `ScheduleEntry`.
**Целевая аудитория:** Завуч / Диспетчер расписания.

## 1. Философия интерфейса
Интерфейс решает две фундаментальные задачи:
1. **Сбор "хотелок" и ограничений** (Шаблоны и больничные/отгулы) для алгоритма генерации расписания (FET).
2. **Диспетчеризация "на лету"** (Поиск окон для замен или общих совещаний).

Главная проблема классических календарей — невозможность отличить "учитель свободен, потому что его нет в школе" от "учитель свободен, потому что у него сейчас окно между уроками". Данный UI решает эту проблему через внедрение **5-мерной матрицы состояний** и **мини-диаграмм (Стек-баров)**.

---

## 2. Глобальный Layout (Split View)
Страница разделена на две части по принципу Master-Detail:

### 2.1 Левая панель (30% ширины): Справочник и Селектор
Инструмент для быстрого поиска нужной группы людей:
*   **Поиск**: Поле ввода по ФИО.
*   **Smart-фильтры (Теги)**: Быстрый выбор кафедр (например, `[Все математики]`, `[Началка]`).
*   **Список учителей**: Выводится в виде карточек с **чекбоксами**. Завуч может выбрать одного или нескольких учителей одновременно.
*   **Индикаторы**: Беджи возле ФИО, если есть активный `Override` (например, 🤒 "Больничный").

### 2.2 Правая панель (70% ширины): Анализатор
*   **Шапка**: Селектор недель (например, `< 15 - 21 Сентября >`). По умолчанию выбрана текущая неделя.
*   **Кнопка переключения**: Тумблер `[Режим просмотра] / [Режим редактирования шаблона]`. Редактирование доступно только если слева выбран ровно **один** учитель.
*   **Сетка**: Рабочая область в виде непрерывного горизонтального таймлайна (По оси Y — Дни недели: Пн-Пт; По оси X — Непрерывное время с 08:00 до 18:00).

---

## 3. Матрица из 5 состояний (Доменная логика)

В интерфейсе строго разведены понятия *Доступности* (в здании ли учитель) и *Занятости* (ведет ли урок прямо сейчас).

1. ⚪ **Не отмечено / Пусто** (Default)
   * *Смысл*: Неизвестно, ставить уроки наугад не стоит (надо выяснять у учителя).
   * *Визуал*: Никак не отображается (полностью прозрачная зона, отсутствие цвета).
2. 🟢 **Доступен / Предпочтительно** (`AVAILABLE`, `PREFERRED`)
   * *Смысл*: Учитель в здании, готов работать. Идеальное "Окно" (Free).
   * *Визуал*: Ячейка залита светло-зеленым. Это главная ценность для завуча.
3. 🟡 **Непредпочтительно** (`UNAVAILABLE` в мягкой форме)
   * *Смысл*: "Могу, но лучше не ставить".
   * *Визуал*: Светло-желтая ячейка (предупреждение).
4. 🔴 **Четко недоступен** (`UNAVAILABLE` жесткий блок / Отгулы)
   * *Смысл*: Выходной, больничный, неприемные часы. Менять расписание бессмысленно.
   * *Визуал*: Серая или бледно-красная ячейка с **диагональной штриховкой** (///).
5. 🔵 **Занят уроком** (`ScheduleEntry`)
   * *Смысл*: Фактическая нагрузка по расписанию.
   * *Визуал*: Плотная **синяя карточка** поверх фона, внутри текст `10А Алгебра`. Перекрывает собой статусы 1-3.

---

## 4. Режимы просмотра сетки

Визуализация правой панели кардинально меняется в зависимости от количества выбранных учителей (чекбоксов слева).

### 4.1 Режим А: Выделен ОДИН учитель (График человека)
Сетка показывает подробную картину одного преподавателя.
*   Завуч видит, где у человека "Окна" (светло-зеленые зоны без синих карточек).
*   Видит наложения (например, синяя карточка легла на заштрихованную красную зону — ошибка диспетчеризации, подсвечивается красной обводкой карточки).

### 4.2 Режим Б: Выделено НЕСКОЛЬКО учителей (Поиск пересечений)
Киллер-фича для поиска замен или времени для собраний кафедры, учитывающая **непрерывное течение времени** (уроки смещаются, начинаются не всегда в 00 минут).
Система "схлопывает" расписание выбранной группы в многослойный график (Stacked Area Chart) на каждый день.

**Визуал (Диаграмма площади):**
*   Каждый день (например, строка "Понедельник") представляет собой горизонтальный таймлайн.
*   На таймлайне рисуется непрерывный "горный рельеф", который складывается из заданных состояний.
*   **Правило высоты горы**: Каждый выделенный учитель добавляет **+1 единицу высоты** (свой цвет) к общей горе, только если у него есть явно заданный статус (Доступен, Занят, Отгул, Желтый). Если его статус "⚪ Не отмечено", он добавляет **0 единиц высоты** (он прозрачен/отсутствует).
*   *Пример*: Выбрано 5 человек. В 10:00 двое зеленых, один синий, двое вообще ничего не заполнили. Общая высота рельефа будет 3 (а не 5), состоящая из зеленого и синего слоев. В 10:15 один не заполнивший приходит на урок (синий статус) — график по оси Y вырастает до 4.
*   Чем "выше" и плотнее гора — тем больше полнота данных. Если гора плоская, значит расписание и доступность никем не заполнены.

**Взаимодействие (Hover Playhead):**
*   Завуч водит мышкой влево-вправо вдоль дня.
*   За курсором тянется **вертикальная линия-визир** (Playhead), как в программах видеомонтажа или биржевых дашбордах.
*   Мгновенно всплывает Tooltip, показывая "срез" состояний точно в нужную секунду:
> **10:23**  
> **Свободны (2)**: Иванова, Смирнов  
> **На уроках (2)**: Петров (10А), Сидоров (11Б)  
> **Недоступен (1)**: Кузнецов (Отгул)

## 5. Редактирование доступности (Только Режим А)

Массовое редактирование (`Bulk Edit`) **запрещено**, чтобы не сломать алгоритм случайным кликом. Завуч вносит изменения только индивидуально.

**Модальное окно (или Sheet) Редактирования:**
Вызывается кнопкой `Редактировать доступность` в профиле выбранного учителя (Слева).
Имеет 2 вкладки:
1.  **Базовый шаблон (`TeacherAvailability`)**:
    *   Сетка 5x10 (Только рабочие часы).
    *   Режим "Кисточки": Завуч выбирает цвет (Зеленый/Желтый/Красный) и кликом "закрашивает" ячейки. Изменения применяются ко всем будущим неделям.
2.  **Исключения (`TeacherAvailabilityOverride`)**:
    *   Табличный список. Нажатие `+ Добавить исключение`.
    *   Форма: `Дата От` - `Дата До` - `Тип (Например, Больничный)`. Это мгновенно наложит "красную штриховку" на выбранные даты в календаре.

---

## 6. Компоненты UI (shadcn/ui ожт Libraries)

*   `Checkbox` — для мультиселекта в левой панели.
*   `Tooltip` — важнейший компонент для расшифровки Стек-бара при наведении.
*   `recharts` (AreaChart) или кастомный SVG/Canvas — для отрисовки непрерывного многослойного "горного рельефа" с линией-визиром (Playhead).
*   `Dialog` / `Sheet` — для формы редактирования исключений.
*   `ScrollArea` — для длинного списка учителей в левой панели.
