# Спецификация UI/UX: Управление помещениями (`/rooms`)

Этот документ описывает детальный интерфейс и пользовательские сценарии для модуля "Здания и Кабинеты".

## 1. Контекст и Данные
Оперируемые сущности БД:
- **`Building`** (Здание): `id`, `name` (напр. "Главный корпус"), `address`.
- **`Room`** (Кабинет): `id`, `name` (напр. "Кабинет 302", "Спортзал"), `capacity` (вместимость), `buildingId` (привязка к зданию).
- **`RoomSubject`**: Предметы, которые *разрешено* вести в этом кабинете (напр. Химия только в кабинете с вытяжкой).

---

## 2. Макет страницы (Layout)

Страница использует паттерн **Master-Detail (Слева меню, справа контент)**, который идеально подходит для иерархии "Здание → Кабинеты".

### Левая колонка: Навигация по Зданиям (Master)
Узкая колонка (ширина ~300px).
- **Заголовок**: "Здания" и рядом кнопка-иконка `[+]` (Добавить здание).
- **Список**: Вертикальное меню зданий. Выбранное здание подсвечивается (Active State).
- Раздел "Все здания" (показывает кабинеты со всех корпусов).

### Правая колонка: Сетка Кабинетов (Detail)
Широкая рабочая область.
- **Header**: Название выбранного здания (например, "Главный корпус: 42 кабинета"). Рядом кнопка `[+ Добавить кабинет]`.
- **Панель инструментов**: Строка поиска по названию кабинета, фильтр по вместимости (например, "> 30 мест").
- **Контент**: Таблица (`TanStack Table`) со столбцами:
  1. `Название` (жирный текст)
  2. `Вместимость` (число)
  3. `Доп. Предметы` (Список бейджей/тегов, напр. [Химия] [Физика])
  4. `Действия` (Меню `...` -> Редактировать / Удалить)

---

## 3. Empty States (Пустые состояния)

### А. Нет ни одного здания в системе
- На весь экран по центру красивая SVG-иллюстрация (например, пустая школа).
- Текст: "Создайте первое здание. Для добавления кабинетов необходимо сначала добавить хотя бы один учебный корпус."
- Большая кнопка: `[+ Добавить здание]`.

### Б. Нет кабинетов в выбранном здании
- Левая колонка показывает список зданий. Правая пустая.
- По центру правой колонки иконка коробки/двери.
- Текст: "В этом здании пока нет кабинетов".
- Кнопка: `[+ Добавить первый кабинет]`.

---

## 4. Поток создания Здания (Building Creation Flow)

Запускается по кнопке `[+]` в левой колонке.
- **Тип UI**: `Dialog` (Модальное окно по центру).
- **Тайтл**: Добавить новое здание.
- **Форма** (`TanStack Form` + `Zod`):
  1. `Название` (Текст, обязательно, мин. 3 символа, макс. 50).
  2. `Адрес` (Текст, опционально, макс. 100).
- **Кнопка**: `[Сохранить]`. При клике - состояние "Loading" (спиннер на кнопке, поля блокируются).
- **Успех**: Диалог закрывается, в левом списке мгновенно появляется новое здание, Toast: ✅ "Здание 'Главный корпус' успешно добавлено".

---

## 5. Поток создания Кабинета (Room Creation Flow)

Запускается по кнопке `[+ Добавить кабинет]` в правой колонке.
- **Тип UI**: Классическое модальное окно (`Dialog`).
- **Тайтл**: Добавить кабинет.
- **Форма**:
  1. `Здание` (`Select`, по умолчанию активное здание из меню).
  2. `Название/Номер` (Текст, обязательно. Например "Актовый зал" или "214").
  3. `Вместимость` (Number Input, по умолчанию `30`, обязательно `> 0`).
- **Кнопки**:
  - `[Сохранить]` — просто закрывает диалог. Тост: "Кабинет добавлен".
  - `[Сохранить и настроить предметы]` — основной призыв к действию (Primary Button). Сохраняет кабинет и сразу переводит в интерфейс "Назначение предметов".

---

## 6. Интерфейс "Сопоставление Предметов" (The Tactile Buckets)

Вызывается после создания кабинета (через "Сохранить и настроить") или из меню кабинета `... -> Настроить предметы`.
- **Тип интерфейса**: Полноэкранное модальное окно (Full-screen Dialog) или широкая выезжающая панель (`Sheet`), чтобы дать максимум пространства.
- **Визуальная метафора**: Паттерн *Transfer List* (Два ведра/колонки).
- **Левая колонка**: «Доступные предметы». 
  - Панель поиска предметов.
  - Предметы выглядят как кликабельные карточки (Pills).
- **Правая колонка**: «Разрешено в Кабинете 302» (Пустая по умолчанию).
- **Механика взаимодействия**:
  - Кабинет ощущается "физическим": администратор использует **Drag-and-Drop** для перетаскивания предметов из левой колонки в правую.
  - Альтернатива для планшетов: на каждой карточке слева есть кнопка `[+]`, справа `[-]`. При нажатии карточка с плавной анимацией "перелетает" в соседнюю колонку.
  - Сохранение происходит мгновенно в фоне (optimistic UI) при каждом перетаскивании/нажатии, без необходимости нажимать общую кнопку "Сохранить".

---

## 7. Редактирование и Защита от удаления (Edit / Delete Flow)

- **Редактирование настроек**: Вызывается из меню `... -> Редактировать`. Открывает `Dialog` с базовыми полями (Название, Вместимость).
- **Удаление кабинета**: Вызывается из меню `... -> Удалить`.
- **Защита от дурака**:
  - Открывается `AlertDialog` (Красный цвет): "Вы уверены? Это действие нельзя отменить."
  - **Блокировщик БД**: Если кабинет УЖЕ задействован в составленном расписании (`ScheduleEntry`), удалить его нельзя! Кнопка "Удалить" отключена (или сервер вернет ошибку в Toast: ❌ "Невозможно удалить кабинет: он используется в текущем расписании").
