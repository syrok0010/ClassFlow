# Спецификация UI/UX: Учебный план / Требования (`/requirements`)

Этот документ описывает интерфейс "Матрицы Нагрузки". Это центральный экран системы, где формируется программа: кто, что и сколько часов учит. От качества UX этого экрана зависит, будет ли завуч любить или ненавидеть систему.

---

## 1. Контекст и Данные
Оперируемые сущности БД:
- **`GroupSubjectRequirement`** (Требование к предмету для группы): 
  - `groupId` (напр. Класс 10А)
  - `subjectId` (напр. Алгебра)
  - `weeklyLessons` (кол-во уроков в неделю, напр. 4)
  - `durationMinutes` (длительность урока, по умолчанию 45)
  - `breakDuration` (перемена после урока, по умолчанию 0)

---

## 2. Глобальный Layout (Excel-подобный Spreadsheet)

Интерфейс строится по паттерну "Бесконечная электронная таблица" (`Pivot Table` / `Data Grid`). 

### Шапка и Фильтры
- Левый сайдбар сворачивается до иконки (mini-sidebar), отдавая 95% ширины экрана под таблицу.
- Панель фильтров плавающая (sticky):
  - `Срез по дисциплинам`: [Все предметы] / [Только Академические] / [Только Допы (Electives)].
  - `Срез по классам`: [Все классы] / [Только начальная школа].
  - **Киллер-фича `[Режим Быстрого Ввода]`**: Тумблер (Switch). Если включен, ячейки при клике просят ввести ТОЛЬКО количество часов (используя стандартные 45 минут), скрывая настройки перемен. Умножает скорость ввода на 10.

### Архитектура Сетки
- **Первый столбец (Ось Y - Заморожен)**: Список групп и классов. Древовидная структура (Tree Data).
  - Уровень 1: `1А`, `1Б`, `Робототехника`.
  - Уровень 2 (скрыт под стрелочкой): `1А Английский (Гр 1)`, `1А Английский (Гр 2)`.
- **Остальные столбцы (Ось X — Предметы)**: Имеют **двухуровневый заголовок** (Column Grouping) для предотвращения визуального перегруза.
  - **Верхний уровень (Группы)**: 3 магистральных блока: `[Основные предметы]`, `[Общие допы]`, `[Допы по выбору]`. 
  - **Сворачивание колонок**: У каждого глобального заголовка есть кнопка `[-]`. При нажатии вся группа предметов (например, 15 колонок "Допов") сворачивается в одну узкую колонку-заглушку `[+] Общие допы`, освобождая экранное пространство для работы с основными предметами.
  - **Нижний уровень (Предметы)**: Непосредственно дисциплины (Математика, Бассейн, Робототехника). Под названием предмета выводится мелкая цифра: `Всего: 45 ч.` (сумма введенных часов по всей школе, полезно для тарификации).
- **Самый правый столбец (Ось Y - Заморожен)**: **Итого часов на класс**.

## 3. Магия Ячейки (The Cell) и Навигация

Ячейка (Intersection Класса и Предмета) — это микро-приложение.

### 3.1. Визуальные состояния
1. **Empty (Пустая)**: Бледно-серая. Означает, что предмета нет. При наведении появляется полупрозрачный `[ + ]`.
2. **Filled (Заполненная)**: 
   - Крупно: **4** (Часа в неделю).
   - Мелко внизу: `45м` (Длительность).
   - Фон слегка подсвечен цветом типа предмета (синий для академических, оранжевый для допов).
3. **Editing (Редактирование)**: Ячейка приподнимается над сеткой (Shadow). Внутри 1 или 3 поля ввода (в зависимости от `Режима Быстрого Ввода`).

### 3.2. Жесткая клавиатурная навигация (Keyboard-First UX)
Суть: Завуч не должен трогать мышь.
- `Клик` по ячейке или перемещение к ней стрелочками `[↑][↓][←][→]` (фокус как в Excel).
- `Enter` или "ввод цифры" на активной ячейке: Переход в режим `Editing`.
- **Сохранение и переход**:
  - Находясь в режиме редактирования:
    - Нажатие `Enter`: Сохраняет данные (Optimistic UI) и прыгает **к ячейке НИЖЕ** (вводить математику для следующего класса).
    - Нажатие `Tab`: Сохраняет данные и прыгает **к ячейке СПРАВА** (вводить русский для этого же класса).

### 3.3. Массовое копирование (Drag-to-Fill)
На заполненной ячейке в правом нижнем углу есть "точка" (как в Excel/Google Sheets).
- Завуч зажимает точку мынкой и тянет вниз (на классы `1Б`, `1В`, `1Г`).
- Отпускает.
- Система мгновенно отправляет батч-запрос (Bulk Create / Update) к серверу, проставляя везде одинаковые настройки предмета.

## 4. Специфичные сценарии (Corner Cases)

### Сценарий 1: Распределение часов на Подгруппы
В строке `10А` в колонке "Английский" стоит прочерк (`-`), который нельзя редактировать.
Почему? Потому что Английский ведется по подгруппам.
Завуч раскрывает дерево `10А`:
- Подгруппа `10А Англ. 1`: Ставит 3 часа.
- Подгруппа `10А Англ. 2`: Ставит 3 часа.
Если завуч попытается поставить 3 часа в сам `10А`, система выдаст Toast: *"Невозможно. Сначала удалите часы из подгрупп"*.

### Сценарий 2: Тепловая карта перегрузок (Heatmap)
Последняя колонка "**Итого часов на класс**" суммирует всю введенную по строке нагрузку.
- Если сумма находится в пределах СанПиН (например, до 32 часов), она зеленая.
- Если сумма становится равна 34, ячейка желтеет.
- Если 36, ячейка становится ярко-красной.
Это позволяет администратору на лету видеть, что он "перегрузил" пятиклассников.

### Сценарий 3: Массовое удаление (Wipe)
Чтобы очистить часы:
- Завуч выбирает ячейку мышью (или выделяет несколько ячеек через Shift+Click).
- Нажимает `Delete` или `Backspace`.
- Данные мгновенно стираются (удаление записей `GroupSubjectRequirement` в БД).

## Резюме для Разработчика
- Инструменты: `TanStack Table` (виртуализация строк и колонок обязательна, так как 50 классов * 40 предметов = 2000 DOM элементов), `TanStack Form` (для ячеек).
- Мутации: Через Server Actions с `useMutation` (TanStack Query) для поддержки Optimistic Updates (чтобы интерфейс не "подвисал" при сохранении каждой ячейки).
