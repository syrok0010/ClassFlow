# Спецификация UI/UX: Глобальный Layout (Dashboard)

Этот документ описывает общую структуру интерфейса приложения ClassFlow, которая оборачивает все внутренние страницы.

## 1. Архитектура Layout
Мы используем современную и минималистичную структуру **Side Navigation (Sidebar) + Content Area** без визуально тяжелого верхнего Header'а.
Это смелый паттерн, который экономит 60px бесценного вертикального пространства экрана — оно нам пригодится для огромных таблиц (Матрицы Учебного плана, Расписания), отдавая под их рендер 100% высоты окна.

---

## 2. Глобальный Sidebar (Левое меню навигации)
Sidebar берет на себя абсолютно все функции: и навигацию, и управление аккаунтом. 
Он занимает левую часть экрана (ширина ~240px). 
**Механика сворачивания (Expand-on-hover)**: На сложных многоколонных экранах вроде Учебного плана Sidebar по умолчанию свернут до узкой полоски иконок (~64px). В свернутом состоянии в его шапке (вместо логотипа) отображается **Гамбургер-меню**. При клике или просто при *наведении мыши* (Hover), Sidebar элегантно расширяется поверх контента, не сдвигая саму таблицу.

### Архитектура Sidebar'а (Role-Based Access)

Меню формируется динамически в зависимости от ролей пользователя. Если человек является одновременно Учителем и Родителем, он увидит обе соответствующие группы в меню. Это позволяет изолировать контекст.

**Группа 1: Администрирование (Видят только Admins)**
*Инструменты конфигурации и составления расписания.*
- ⚙️ **Дашборд школы** (`/admin`) - Сводка.
- 📆 **Редактор расписания** (`/admin/schedule`) - *Генератор v0.4*.
- 🏢 **Инфраструктура** (`/admin/rooms`, `/admin/subjects`) - Справочники.
- 👥 **Контингент** (`/admin/groups`, `/admin/users`) - Люди и классы.
- 📊 **Учебный план** (`/admin/requirements`) - Матрица нагрузки.

**Группа 2: Мое преподавание (Видят только Teachers)**
*Рабочий стол педагога.*
- 📚 **Мои предметы** (`/teacher/subjects`) - Какие предметы и для каких классов ведет преподаватель.
- 📅 **Мое расписание** (`/teacher/schedule`) - Уроки на неделю.
- 🙋‍♂️ **Моя доступность** (`/teacher/availability`) - Запрос тайм-слотов у завуча.

**Группа 3: Мои дети (Видят только Parents)**
*Раздел для родителей.*
- 👶 **Дневники детей** (`/parent/children`) - Выбор ребенка для просмотра его расписания.

**Группа 4: Моя учеба (Видят только Students)**
*Раздел для учеников.*
- 🎓 **Расписание уроков** (`/student/schedule`).

**Футер Sidebar (Профиль Пользователя):**
В самом низу Sidebar'а закреплен личный кабинет текущего пользователя. Это выглядит как большая кнопка-карточка:
- **Слева**: Круглый аватар (с фото или инициалами).
- **Справа (если Sidebar развернут)**: Имя пользователя и мелким шрифтом его роль или краткое представление нескольких ролей.
  Если пользователь совмещает несколько доменных ролей, подпись не должна притворяться, что роль одна.
- По клику на этот профиль открывается всплывающее меню (`Popover` / `DropdownMenu` из shadcn) вправо:
  1. Настройки профиля.
  2. Переключатель темы (Светлая / Темная).
  3. Раздел "Помощь".
  4. Красная кнопка `[Выйти из системы]`.

## 3. Область Контента (Main Content Area)
Это то место, куда рендерятся сами страницы (`app/(dashboard)/layout.tsx` -> `{children}`).
Поскольку глобального хедера нет, страницы сами должны заботиться о своих заголовках. Вверху каждой страницы (над заголовком `H1`) опционально располагаются **Хлебные крошки (Breadcrumbs)** для ориентации (например: `Академическая структура / Помещения`).

- **Фон**: Светло-серый (например, `bg-slate-50` из Tailwind) для отделения контента от белого Sidebar и карточек.
- **Отступы (Padding)**: Стандартные `p-6` или `p-8` по бокам. НА сложной странице "Учебного плана" отступы убираются (`p-0`), чтобы таблица заняла 100% ширины.
- **Глобальный Toaster**: В этой области всегда невидимо висит компонент `<Toaster />` от библиотеки `sonner`, который выкидывает красивые уведомления в правом нижнем углу.
