# Спецификация UI/UX: Управление пользователями (`/admin/users`)

Этот документ описывает интерфейс и сценарии для модуля управления персоналом и учениками. Это "Центр управления людьми", доступный только администраторам.

## 1. Контекст и Данные
Оперируемые сущности БД:
- **`User`** (Учетная запись): `id`, `name`, `email`, `role` (Системная роль: `ADMIN` или `USER`), `status` (Статус аккаунта: `ACTIVE`, `PENDING_INVITE`, `DISABLED`).
- **Связанные доменные роли** (существование записи в этих таблицах определяет, кто этот человек в школе):
  - **`Teacher`** (Учитель).
  - **`Student`** (Ученик).
  - **`Parent`** (Родитель).

*Один `User` может быть одновременно Учителем и Родителем (например, Мария Ивановна ведет математику, а её сын учится в 5А).*

---

## 2. Макет страницы (Layout)

Классический паттерн **Data Grid** на всю ширину рабочей области (Main Content Area).

### Header (Шапка страницы)
- Заголовок `H1`: "Пользователи".
- Большая основная кнопка (Primary Button): `[+ Добавить пользователя]`.

### Панель фильтров (Toolbar)
Располагается над таблицей.
- Поисковая строка (`Input` с иконкой лупы): Поиск по имени или E-mail.
- Фильтр по Доменной роли (`Select` или группа кнопок `ToggleGroup`): `Все`, `Учителя`, `Ученики`, `Родители`.
- Фильтр по Статусу (`DropdownMenu` с чекбоксами): `Активные`, `Ожидают инвайт`.

### Таблица Пользователей (`TanStack Table`)
- **Колонки**:
  1. `Пользователь` (Аватар + Имя + `email` мелким серым текстом под именем).
  2. `Системная роль` (Бейдж: Синий `Admin` / Серый `User`).
  3. `Роли в школе` (Стек бейджей: Желтый `Учитель`, Зеленый `Ученик`, Фиолетовый `Родитель`).
  4. `Статус` (Зеленая точка `Активен` / Желтая точка `Ожидает: отправлен 2 дня назад`).
  5. `Действия` (Меню `...`).

---

## 3. Empty States (Пустые состояния)

### А. Нет результатов поиска/фильтрации
- По центру таблицы иконка лупы.
- Текст: "Пользователи не найдены".
- Кнопка-ссылка: "Сбросить фильтры".

## 4. Поток создания пользователя (Smart Row / Inline Edit)

Для обеспечения максимальной скорости заведения новых пользователей (что критически важно перед началом учебного года), мы отказываемся от громоздкой выезжающей панели (Sheet) в пользу **потокового редактирования прямо в таблице (Inline Edit / Smart Row)**.

Запускается по кнопке `[+ Добавить пользователя]`.

### 4.1 Конвейер создания (Ученики, Учителя, Админы)
1. При нажатии на кнопку в самом верху таблицы (сразу под заголовками колонок) появляется **новая интерактивная строка**.
2. Администратор делает всё, не отрывая рук от клавиатуры:
   - Вводит ФИО (обязательно) -> нажимает `Tab`.
   - Вводит `Email` (опционально, можно оставить пустым для получения кода) -> нажимает `Tab`.
   - Выбирает основную роль из выпадающего списка (`Ученик`, `Учитель`, `Администратор`) -> нажимает `Enter`.
3. **Магия конвейера (Optimistic UI)**:
   - Строка мгновенно переходит в режим чтения (Ready).
   - В колонке действий (Action) появляется кнопка **`[Скопировать инвайт-ссылку]`** (если Email не был указан).
   - **Сразу же под сохраненной строкой автоматически появляется новая пустая интерактивная строка.**
   - Администратор продолжает вбивать список, пока не нажмет `Esc` или не кликнет мышкой в свободное место.

*Почему это круто: Mouse-free UX. Заведение целого класса из 30 человек превращается в непрерывную штамповку строк в стиле мощного Excel.*

### 4.2 Поток генерации/Связывания "Родителя"
Для родителей используется принципиально иной сценарий: классный руководитель или админ **не должен** вручную вбивать ФИО или почту мам и пап. Вместо этого он генерирует связь со стороны ребенка.
- В таблице администратор находит строку с нужным учеником (например, "Иванов Петя").
- Кликает на меню действий `...` в конце строки и выбирает **`[Сгенерировать инвайт для родителя]`**.

**Окно "Сгенерировать инвайт" (Dialog)**:
Появляется модальное окно с двумя вкладками (Tabs):
1. **Вкладка "Новый родитель" (По умолчанию)**:
   - Система возвращает уникальный код (например, `X7B9-A2MZ`).
   - Кнопка: `[Скопировать и отправить в мессенджер]`. Этот код создаст пустую учетку (см. пункт 4.3).
2. **Вкладка "Существующий родитель" (Брат/Сестра)**:
   - Поисковая строка: "Начните вводить ФИО родителя...".
   - Администратор находит "Иванов Иван Иванович" (который уже зарегистрирован в школе, так как его старшая дочь уже учится в 10 классе).
   - Выбирает его из выпадающего списка и жмет `[Связать]`. 
   - Никаких инвайтов не отправляется, в профиле Ивана Ивановича просто появляется второй ребенок.

### 4.3 Поток активации (На стороне Ученика/Родителя)
Когда ученик или родитель переходит по сгенерированной ссылке `.../invite/X7B9-A2MZ`:
1. Он видит приветственный экран ("Добро пожаловать в систему ClassFlow").
2. Форма просит его **самостоятельно** вбить или исправить свои контактные данные:
   - Если это Учитель/Ученик, он видит введенные админом ФИО и может их скорректировать.
   - Если это Родитель, он видит чистые поля `Имя`, `Фамилия`, `Email` и обязан их заполнить.
3. Пользователь придумывает `Пароль`.
4. После сабмита статус в базе меняется на `ACTIVE`, и у родителя появляется привязка к дневнику своего ребенка.

## 5. Редактирование и Управление (Action Menu)

Клик по `...` в конце строки таблицы открывает Dropdown-меню:

1. **Редактировать профиль** -> Открывает панель `Sheet`. 
   Здесь происходит **управление дополнительными ролями (Cross-role assignment)**:
   - Если это Учитель (например, Марья Ивановна), админ может в Секции 3 (Роли) дополнительно поставить чекбокс `[x] Родитель`.
   - При сохранении появится связанный блок: "Укажите детей этого пользователя" (поиск `Student` из базы для привязки).
   - Аналогично, если уборщица (User) стала вести кружок, ей можно просто доставить галочку `[x] Учитель`.
2. **Скопировать ссылку-инвайт** *(видно только для статуса PENDING)* -> Копирует токен авторизации в буфер обмена (если email не дошел и нужно скинуть в WhatsApp).
3. **Заблокировать доступ** -> Меняет статус на `DISABLED`. Строка в таблице становится полупрозрачной. Человек не сможет зайти в систему.
4. **Удалить пользователя** (Красный цвет) -> Запускает `AlertDialog` "Вы уверены? Будут удалены все связи (оценки, расписание)." -> Требует ввести имя пользователя для подтверждения удаления (Hard Deletion Protection).
