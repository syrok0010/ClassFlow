# Заметки по Playwright

- Тесты запускаются на изолированной БД из `DATABASE_URL_E2E`; использовать локальную dev-БД категорически нельзя, потому что перед каждым прогоном данные пересоздаются.
- В `globalSetup` всегда выполняются `prisma db push` и `prisma/seed.e2e.ts`, поэтому тестовые фикстуры считаются частью контракта тестов.
- Авторизация заранее создается в `tests/e2e/auth.setup.ts` и переиспользуется через `tests/e2e/.auth/admin.json`; это убирает флейки от повторных логинов и стабилизирует prod-like прогон.
- Для e2e встроенный rate limit Better Auth отключается через env у Playwright `webServer`, иначе частые повторные логины начинают давать ложные `429 Too Many Requests`.
- В `auth.spec.ts` общий `storageState` намеренно отключен, чтобы проверять редирект для неавторизованного пользователя и сам сценарий логина.
